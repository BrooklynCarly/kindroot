# Product Requirements Document: Parent Quiz Website

## Overview
A single-page, mobile-first website designed to help parents support each other through an engaging quiz experience. The site emphasizes authenticity and peer support over commercial branding.

**Website URL:** kindroot.io  
**Quiz URL:** https://form.typeform.com/to/sf98mAlp

## Product Objectives
- Create a welcoming, grassroots platform that feels like parents helping parents
- Drive quiz engagement through clear, prominent call-to-action
- Maintain simplicity and authenticity in design and messaging
- Ensure seamless experience across all mobile devices

## Target Audience
**Primary**: Parents and caregivers seeking peer support and information
- Mobile-first users (80%+ expected mobile traffic)
- Looking for quick, accessible resources
- Value authentic, non-commercial advice

## User Stories
1. As a parent on mobile, I want to quickly understand what the quiz offers so I can decide if it's relevant to me
2. As a visitor, I want to easily start the quiz without navigating multiple pages
3. As a skeptical user, I want to understand why this resource exists and who created it
4. As a mobile user, I want smooth scrolling and readable text without zooming

## Core Features & Requirements

### 1. Hero Section with Primary CTA
**Must Have:**
- Large, prominent "Take the Quiz" button
- Clear, concise headline explaining value proposition
- Button should be visually dominant (but not aggressive)
- Mobile: Button should be thumb-friendly (minimum 44x44px tap target)
- **Button links to:** https://form.typeform.com/to/sf98mAlp

**Content Requirements:**
- Headline: One clear sentence about the quiz's purpose
- Optional: Short supporting text (1-2 sentences max)
- CTA copy: "Take the Quiz" or similar action-oriented text

### 2. About/Why Section
**Must Have:**
- Authentic explanation of purpose
- Written in warm, conversational tone
- Explains the "parents helping parents" mission
- No corporate language or sales pitch

**Content Requirements:**
- Why the quiz exists
- Who created it (real people, not brand)
- What users can expect
- Keep it genuine and relatable

**Design Requirements:**
- Should feel like a personal note, not marketing copy
- Consider using personal photos or simple illustrations
- Avoid stock imagery

### 3. Design & Visual Identity

**Design Principles:**
- Simple, clean, uncluttered
- Grassroots aesthetic - not overly polished
- Friendly but not childish
- Accessible and readable

**Visual Guidelines:**
- Minimal color palette (2-3 colors max)
- Plenty of white space
- Simple typography (1-2 fonts)
- Avoid: gradient backgrounds, complex animations, corporate design elements
- Prefer: clean layouts, honest imagery, straightforward presentation

### 4. Mobile-First Requirements

**Critical Mobile Considerations:**
- Touch-friendly interactive elements (minimum 44x44px)
- Readable font sizes (minimum 16px body text)
- Fast load time (target: under 3 seconds on 3G)
- No horizontal scrolling
- Responsive images
- Single column layout on mobile

**Breakpoints:**
- Mobile: 320px - 767px (primary focus)
- Tablet: 768px - 1024px
- Desktop: 1025px+

### 5. Technical Requirements

**Performance:**
- Lightweight page (target: under 1MB total)
- Optimized images
- Minimal external dependencies
- Fast time to interactive

**Browser Support:**
- iOS Safari (last 2 versions)
- Chrome Mobile (last 2 versions)
- Chrome Desktop (last 2 versions)
- Firefox (last 2 versions)
- Edge (last 2 versions)

**Accessibility:**
- WCAG 2.1 Level AA compliance
- Semantic HTML
- Keyboard navigation support
- Screen reader friendly
- Sufficient color contrast (4.5:1 minimum)

## Page Structure

### Layout Flow (Top to Bottom)
1. **Hero Section**
   - Headline
   - Primary CTA button
   - Optional: Brief supporting text

2. **About/Why Section**
   - Heading
   - Personal explanation (2-3 paragraphs)
   - Optional: Photo or simple visual

3. **Secondary CTA** (Optional)
   - Repeat quiz button for users who scrolled past hero
   - Links to same Typeform: https://form.typeform.com/to/sf98mAlp

### Content Guidelines
- Use conversational, first-person language
- Avoid jargon and formal business speak
- Keep paragraphs short (2-4 sentences)
- Use natural, authentic voice
- No sales pressure or hype

## Out of Scope (Not Included)
- Multiple pages or complex navigation
- User accounts or login
- Elaborate animations or interactive elements
- Email capture forms (unless explicitly added later)
- Social media integration
- Commercial branding elements
- Marketing analytics beyond basic page views

## Success Metrics
- Quiz completion rate
- Mobile vs. desktop traffic ratio
- Page load time
- Scroll depth
- CTA click-through rate

## Launch Requirements
**Before Launch:**
- Mobile testing on actual devices (iOS + Android)
- Load time optimization
- Accessibility audit
- Content review for tone and clarity
- Cross-browser testing

## Future Considerations (Post-Launch)
- A/B testing different CTA copy
- Optional email opt-in
- Quiz results sharing capability
- Testimonials or user feedback section

## Key Stakeholder Notes
- Maintain the "parents helping parents" authenticity above all else
- Resist temptation to over-design or add commercial elements
- Keep mobile experience as the priority in all decisions
- When in doubt, choose simplicity
